# XiAgent UI V1 开发子规则

## 适用范围

本文件记录 `ui/V1` 的版本特有 UI 规则。通用规则以 `.codex/skills/xiagent-ui-development/SKILL.md` 为准；本文件只记录 V1 的风格、结构、测试命令和当前版本约束。

## 技术栈

- React 19、TypeScript、Vite、Vitest、Testing Library、Playwright。
- 入口位于 `ui/V1/src/app/App.tsx`，样式主要在 `ui/V1/src/styles/app.css`。
- API 调用集中在 `ui/V1/src/api/`，不得绕过 API client 直接访问后端内部实现。

## 页面与信息架构

- V1 保留项目总览、资产库、任务/工作流相关页面，是可运行 UI 版本，不是设计素材目录。
- `ui/V1/pencli/` 下的 `.pen` 文件可作为 V1 页面视觉参考，但不能替代真实页面和测试。
- 页面应围绕项目、资产、任务和工作流运行闭环组织，不做拖拽式工作流编辑器。

## 视觉规则

- V1 当前偏产品后台/项目总览风格，主色为深紫 `#1f108e`，辅助蓝为 `#175cd3`、`#eef4ff`。
- 背景以 `#f7f9fb` 和白色面板为主，边线使用 `#d8dce6` 系列。
- 卡片、面板、按钮圆角一般为 6-8px；状态和 chip 可使用 999px 胶囊圆角。
- 页面布局偏 dashboard/table/detail：表格、筛选、详情侧栏、任务卡片应保持信息密度，不做营销式大 hero。

## 数据与交互规则

- 用户可见内容必须来自 API、当前状态或明确的表单输入；业务列表不能用静态数组冒充真实数据。
- 工作流契约、节点快照和任务事件只能转换成用户能理解的字段、状态、卡片和操作，不直接展示原始 JSON。
- 资产展示应显示文件、文字、标签、项目/全局范围和可用预览；不得暴露本地存储路径。

## 节点与控件规则

- 如果 V1 继续扩展节点输入/输出 UI，应建立或复用版本内控件注册机制，不把某个工作流的特殊展示硬编码进页面主流程。
- 控件输出必须匹配工作流输入 schema 或人工节点 output schema。

## 验证命令

在 `ui/V1` 下优先运行：

```powershell
npm run test
npm run build
npm run test:e2e
```

涉及后端契约时，同时运行相关 `pytest`。最终验收仍需用真实浏览器走登录、项目选择、目标页面操作和持久化结果确认。

## 维护要求

- 修改 V1 配色、布局密度、页面导航、控件注册或测试策略时，同步更新本文件。
- 从 V2 回迁功能时，先确认 V1 的视觉和信息架构是否仍适用，不要直接复制 V2 样式。
